The goal of this gcode editor is twofold:
1. Allow the user to input a gcode program and quickly gather information about it.
2. Allow the user to quickly make a gcode program more featured and robust.

The user should be able to comfortably access the editor. I believe this can be accomplished through availability on a website, 
with the ability to drag-and-drop the text file onto the page. .txt, .nc, .NC, and .T files should be accepted.

The code that is uploaded can be expected to follow the format described in gcode-structure-initial.txt.
When the code is added, the user should be presented with a text area where they can view/edit the code, and a collapsable panel with a table of information and tools to edit the code. 
There shouldn't be any changes made when the user first uploads a file - optimization features should be toggled off by default.
The table should update in real-time as the user makes changes to the code.
The table should include the information detailed in table.txt.
The panel should include the editing tools listed in editing-features.txt.
